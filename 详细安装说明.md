## 系统要求

### 最低配置
- **操作系统**：Windows 10/11 64位
- **处理器**：Intel i5 或同等性能AMD处理器
- **内存**：8GB RAM（推荐16GB）
- **存储空间**：至少5GB可用空间
- **声卡**：支持音频输入输出的标准声卡

### 推荐配置
- **操作系统**：Windows 10/11 64位
- **处理器**：Intel i7 或更高性能处理器
- **内存**：16GB RAM
- **显卡**：NVIDIA GTX 1060 或更高（支持CUDA）
- **存储空间**：10GB可用空间
- **声卡**：专业音频设备（可选）

## 安装步骤

### 步骤1：下载和解压
1. 下载 `RVC-Sound-Cloning-Out-of-the-Box.zip` 文件
2. 右键点击压缩包，选择"全部解压缩"
3. 选择解压位置（建议解压到非系统盘，如D盘）
4. 点击"解压"按钮等待完成

### 步骤2：检查系统环境
1. 双击运行包内的 `环境检查.bat`
2. 等待检查程序自动检测系统环境
3. 根据检查结果显示进行相应操作：
   - ✅ 绿色提示：环境正常，可直接使用
   - ⚠️ 黄色提示：部分组件需要安装
   - ❌ 红色提示：关键组件缺失，需要安装

### 步骤3：安装缺失组件（如需要）
如果环境检查显示有缺失组件：

1. **安装Python 3.10**：
   - 双击运行 `python-installer.exe`
   - 按照安装向导完成安装
   - 重启计算机

2. **安装依赖库**：
   - 双击运行 `安装依赖.bat`
   - 等待自动安装完成（可能需要5-10分钟）

### 步骤4：首次运行
1. 双击运行 `启动RVC.cmd`
2. 在弹出的菜单中选择功能：
   - 选项1：启动Web训练界面
   - 选项2：启动实时语音转换
   - 选项3：环境检查
   - 选项4：退出程序

## 首次运行配置

### 首次启动Web训练界面
1. 双击 `启动RVC.cmd`

## 功能使用指南

### Web训练界面功能介绍
- **训练标签页**：用于训练音色模型
- **推理标签页**：用于使用训练好的模型进行音色转换
- **UVR5标签页**：用于人声和伴奏分离
- **训练功能标签页**：模型合并、信息修改等功能

### 实时语音转换界面功能介绍
- **模型选择**：选择训练好的音色模型
- **音频设备设置**：配置输入输出设备
- **参数调节**：调整音调、索引比率等参数
- **实时转换**：开始/停止实时语音转换

## 音色训练完整流程

### 第一步：准备音频素材
1. 收集3-30分钟清晰的语音音频
2. 确保音频质量：
   - 无背景噪音
   - 音量适中
   - 发音清晰
3. 将音频文件放入 `datasets/your_voice_name/` 目录
4. 支持的音频格式：WAV、MP3、FLAC、OGG

### 第二步：启动Web训练界面
1. 双击运行 `启动RVC.cmd`
2. 选择选项1启动Web训练界面
3. 在浏览器中打开 `http://localhost:7897`

### 第三步：配置训练参数
1. 点击"训练"标签页
2. 填写基本配置：
   - **实验名称**：输入有意义的名称（如"my_voice"）
   - **目标采样率**：40k（适用于语音）
   - **模型是否带音高指导**：是（推荐）
   - **版本**：v2（推荐）

### 第四步：设置训练文件夹
1. 在"输入训练文件夹路径"中填写音频文件夹路径
   例如：`C:\Users\YourName\RVC-Out-of-the-Box\datasets\my_voice_name`
2. 确认路径正确无误

### 第五步：一键训练
1. 点击"一键训练"按钮
2. 等待系统自动完成以下步骤：
   - 数据预处理
   - 特征提取
   - 模型训练
   - 索引训练
3. 训练进度会实时显示在界面上

### 第六步：使用训练好的模型
1. 训练完成后，进入"模型推理"标签页
2. 在"推理音色"下拉菜单中选择刚训练的模型
3. 上传音频文件进行转换测试

## 实时语音转换使用

### 配置步骤
1. 双击运行 `启动RVC.cmd`
2. 选择选项2启动实时语音转换
3. 在界面中：
   - 选择训练好的模型文件
   - 配置音频输入设备（麦克风）
   - 配置音频输出设备（扬声器）
   - 调整相关参数（音调、索引比率等）

### 使用步骤
1. 点击"开始音频转换"按钮
2. 对着麦克风说话
3. 听到变声后的声音输出
4. 点击"停止音频转换"结束使用

## 常见问题解决

### Q1: 双击启动程序没反应怎么办？
**解决方案**：
1. 右键点击 `启动RVC.cmd`，选择"以管理员身份运行"
2. 检查Windows安全中心是否阻止了程序运行
3. 临时关闭杀毒软件后再次尝试

### Q2: 浏览器无法打开训练界面怎么办？
**解决方案**：
1. 确认程序已正常启动（查看命令行窗口）
2. 尝试使用其他浏览器（Chrome、Edge、Firefox）
3. 检查防火墙设置是否阻止了本地连接
4. 尝试访问 `http://127.0.0.1:7897`

### Q3: 训练过程中出现错误怎么办？
**解决方案**：
1. 截图保存错误信息
2. 检查音频文件路径是否包含中文字符
3. 确认音频文件格式是否支持
4. 检查磁盘空间是否充足
5. 重启程序后重新开始训练

### Q4: 音质不佳怎么办？
**解决方案**：
1. 使用更高质量的原始音频素材
2. 增加训练轮数（total_epoch）
3. 尝试不同的音高提取算法
4. 调整推理时的参数设置

### Q5: 内存不足怎么办？
**解决方案**：
1. 降低batch_size值
2. 减少同时训练的模型数量
3. 关闭其他占用内存的程序
4. 使用CPU训练（较慢但内存占用较少）

### Q6: 训练速度很慢怎么办？
**解决方案**：
1. 确认GPU驱动是否正确安装
2. 降低batch_size值
3. 使用更短的音频片段
4. 考虑使用云端GPU资源

### Q7: 模型文件缺失怎么办？
**解决方案**：
1. 运行 `tools/download_models.py` 下载模型
2. 手动下载模型文件并放入对应目录
3. 检查网络连接是否正常

### Q8: 实时转换有延迟怎么办？
**解决方案**：
1. 使用ASIO音频驱动（如果有）
2. 降低音频缓冲区大小
3. 关闭其他音频处理程序
4. 使用性能更好的音频设备

### Q9: 程序启动时报Python错误怎么办？
**解决方案**：
1. 运行 `环境检查.bat` 检查环境
2. 重新安装Python 3.10
3. 重新安装依赖库
4. 检查系统PATH环境变量

### Q10: 如何获取技术支持？
**解决方案**：
1. 查看终端窗口中的详细错误信息
2. 检查相关日志文件
3. 在社区论坛寻求帮助
4. 提供详细的错误描述和系统信息
