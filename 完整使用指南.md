## 简介

这是一个开箱即用的RVC（Retrieval-based Voice Conversion）音色克隆应用程序包。它包含了运行RVC音色克隆所需的所有文件和依赖，用户可以直接使用而无需复杂的环境配置。

## 系统要求

### 最低配置
- **操作系统**：Windows 10/11 64位
- **处理器**：Intel i5 或同等性能AMD处理器
- **内存**：8GB RAM（推荐16GB）
- **存储空间**：至少5GB可用空间
- **声卡**：支持音频输入输出的标准声卡

### 推荐配置
- **操作系统**：Windows 10/11 64位
- **处理器**：Intel i7 或更高性能处理器
- **内存**：16GB RAM
- **显卡**：NVIDIA GTX 1060 或更高（支持CUDA）
- **存储空间**：10GB可用空间
- **声卡**：专业音频设备（可选）

## 安装步骤

### 步骤1：下载和解压
1. 下载 `RVC-Sound-Cloning-Out-of-the-Box.zip` 文件
2. 右键点击压缩包，选择"全部解压缩"
3. 选择解压位置（建议解压到非系统盘，如D盘）
4. 点击"解压"按钮等待完成

### 步骤2：检查系统环境
1. 双击运行包内的 `环境检查.bat`
2. 等待检查程序自动检测系统环境
3. 根据检查结果显示进行相应操作：
   - ✅ 绿色提示：环境正常，可直接使用
   - ⚠️ 黄色提示：部分组件需要安装
   - ❌ 红色提示：关键组件缺失，需要安装

### 步骤3：安装缺失组件（如需要）
如果环境检查显示有缺失组件：

1. **安装Python 3.10**：
   - 双击运行 `python-installer.exe`
   - 按照安装向导完成安装
   - 重启计算机

2. **安装依赖库**：
   - 双击运行 `安装依赖.bat`
   - 等待自动安装完成（可能需要5-10分钟）

### 步骤4：首次运行
1. 双击运行 `启动应用程序.bat`
2. 在弹出的菜单中选择功能：
   - 选项1：启动实时音色克隆通话（推荐）
   - 选项2：启动Web训练界面
   - 选项3：启动纯命令行应用程序
   - 选项4：退出程序

## 音色训练完整流程

### 第一步：准备音频素材
1. 收集3-30分钟清晰的语音音频
2. 确保音频质量：
   - 无背景噪音
   - 音量适中
   - 发音清晰
3. 将音频文件放入 `datasets/your_voice_name/` 目录
4. 支持的音频格式：WAV、MP3、FLAC、OGG

### 第二步：启动Web训练界面
1. 双击运行 `启动应用程序.bat`
2. 选择选项2启动Web训练界面
3. 在浏览器中打开 `http://localhost:7899`

### 第三步：配置训练参数
1. 点击"训练"标签页
2. 填写基本配置：
   - **实验名称**：输入有意义的名称（如"my_voice"）
   - **目标采样率**：40k（适用于语音）
   - **模型是否带音高指导**：是（推荐）
   - **版本**：v2（推荐）

### 第四步：设置训练文件夹
1. 在"输入训练文件夹路径"中填写音频文件夹路径
   例如：`C:\Users\YourName\RVC-Out-of-the-Box\datasets\my_voice_name`
2. 确认路径正确无误

### 第五步：一键训练
1. 点击"一键训练"按钮
2. 等待系统自动完成以下步骤：
   - 数据预处理
   - 特征提取
   - 模型训练
   - 索引训练
3. 训练进度会实时显示在界面上

### 第六步：使用训练好的模型
1. 训练完成后，进入"模型推理"标签页
2. 在"推理音色"下拉菜单中选择刚训练的模型
3. 上传音频文件进行转换测试

## 实时语音转换使用

### 配置步骤
1. 双击运行 `启动应用程序.bat`
2. 选择选项1启动实时语音转换
3. 在界面中：
   - 选择训练好的模型文件
   - 配置音频输入设备（麦克风）
   - 配置音频输出设备（扬声器）
   - 调整相关参数（音调、索引比率等）

### 使用步骤
1. 点击"开始音频转换"按钮
2. 对着麦克风说话
3. 听到变声后的声音输出
4. 点击"停止音频转换"结束使用

## 常见问题解决

### 安装问题

#### Q1: 双击启动程序没反应怎么办？
**问题描述**：双击 `启动应用程序.bat` 后没有任何反应

**解决方案**：
1. 右键点击 `启动应用程序.bat`，选择"以管理员身份运行"
2. 检查Windows安全中心是否阻止了程序运行
3. 临时关闭杀毒软件后再次尝试
4. 检查文件是否完整解压
5. 确认系统满足最低要求（Windows 10/11 64位）

#### Q2: 运行环境检查脚本报错怎么办？
**问题描述**：运行 `环境检查.bat` 时出现错误

**解决方案**：
1. 确认Python已正确安装
2. 检查系统PATH环境变量是否包含Python路径
3. 重新安装Python 3.10
4. 运行命令提示符（CMD）并输入 `python --version` 确认

#### Q3: 安装依赖时网络超时怎么办？
**问题描述**：运行 `安装依赖.bat` 时出现网络超时错误

**解决方案**：
1. 检查网络连接是否稳定
2. 尝试使用手机热点连接
3. 关闭防火墙和杀毒软件
4. 更换网络环境（如公司网络切换到家庭网络）
5. 手动下载依赖包并离线安装

#### Q4: Python版本不兼容怎么办？
**问题描述**：提示Python版本过低或过高

**解决方案**：
1. 卸载当前Python版本
2. 下载并安装Python 3.10
3. 确保安装时勾选"Add Python to PATH"
4. 重启计算机后重新检查环境

### 运行问题

#### Q5: 浏览器无法打开训练界面怎么办？
**问题描述**：启动Web界面后，浏览器无法访问 `http://localhost:7899`

**解决方案**：
1. 确认程序已正常启动（查看命令行窗口是否有报错）
2. 尝试使用其他浏览器（Chrome、Edge、Firefox）
3. 检查防火墙设置是否阻止了本地连接
4. 尝试访问 `http://127.0.0.1:7899`
5. 重启程序后重新尝试

#### Q6: 实时语音转换界面打不开怎么办？
**问题描述**：启动实时转换界面时程序闪退或无响应

**解决方案**：
1. 运行环境检查脚本确认依赖完整
2. 更新显卡驱动到最新版本
3. 以管理员身份运行程序
4. 检查音频设备是否正常工作
5. 重新安装依赖库

#### Q7: 程序启动时报DLL缺失错误怎么办？
**问题描述**：启动时提示缺少DLL文件（如VCRUNTIME140.dll）

**解决方案**：
1. 下载并安装Microsoft Visual C++ Redistributable
2. 安装Visual Studio 2015-2022 x64版本
3. 重启计算机后重新尝试

### 训练问题

#### Q8: 训练过程中出现CUDA错误怎么办？
**问题描述**：训练时提示CUDA相关错误

**解决方案**：
1. 确认显卡支持CUDA（NVIDIA GTX系列以上）
2. 更新显卡驱动到最新版本
3. 安装对应版本的CUDA Toolkit
4. 降低batch_size值减少显存占用
5. 使用CPU训练（较慢但更稳定）

#### Q9: 训练过程中出现内存不足错误怎么办？
**问题描述**：训练时提示"Out of memory"或类似错误

**解决方案**：
1. 降低batch_size值（默认16，可尝试8或4）
2. 减少同时训练的模型数量
3. 关闭其他占用内存的程序
4. 使用CPU训练（内存占用较少）
5. 增加虚拟内存大小

#### Q10: 训练进度卡住不动怎么办？
**问题描述**：训练进度长时间停留在某个百分比

**解决方案**：
1. 耐心等待，某些步骤可能需要较长时间
2. 检查CPU/内存使用率是否正常
3. 确认硬盘有足够的剩余空间
4. 重启程序后重新开始训练
5. 尝试减少训练数据量

#### Q11: 训练完成后找不到模型文件怎么办？
**问题描述**：训练完成但在推理界面找不到模型

**解决方案**：
1. 检查logs目录下是否有对应的模型文件夹
2. 确认实验名称填写正确
3. 检查文件路径是否包含特殊字符或中文
4. 重新训练并观察文件生成过程

### 音质问题

#### Q12: 转换后的音质很差怎么办？
**问题描述**：转换后的音频有杂音、失真或机械感

**解决方案**：
1. 使用更高质量的原始音频素材
2. 增加训练轮数（total_epoch，推荐200-300）
3. 尝试不同的音高提取算法（RMVPE效果较好）
4. 调整推理时的参数设置：
   - Index Rate: 0.5-0.8
   - Filter Radius: 3
   - Resample: 0
   - Pitch Guidance: 1

#### Q13: 转换后声音听起来像机器人怎么办？
**问题描述**：转换后的音频过于机械化，缺乏自然感

**解决方案**：
1. 增加训练数据量（至少10分钟高质量音频）
2. 调整推理参数：
   - Pitch Guidance: 设置为1
   - Protect Voiceless Consonants: 0.33
3. 尝试不同的预训练模型
4. 使用更长的音频片段进行训练

#### Q14: 转换后声音有回音或混响怎么办？
**问题描述**：转换后的音频有明显的回音效果

**解决方案**：
1. 使用无混响的原始音频进行训练
2. 在推理时调整参数：
   - Reverb: 设置为0
   - Pitch Change (semitones): 0
3. 使用音频编辑软件预处理原始音频

### 硬件问题

#### Q15: 使用GPU训练时报错怎么办？
**问题描述**：启用GPU训练时出现各种错误

**解决方案**：
1. 确认显卡型号支持CUDA（NVIDIA系列）
2. 更新显卡驱动到最新版本
3. 安装对应版本的CUDA和cuDNN
4. 检查显存是否足够（至少4GB）
5. 降低batch_size值减少显存占用

#### Q16: CPU训练速度很慢怎么办？
**问题描述**：使用CPU训练时速度极慢

**解决方案**：
1. 确认CPU支持AVX指令集
2. 关闭其他占用CPU的程序
3. 增加虚拟内存大小
4. 考虑使用云端GPU资源（Google Colab等）
5. 降低训练参数（batch_size、num_workers等）

#### Q17: 音频设备无法识别怎么办？
**问题描述**：实时转换时无法识别麦克风或扬声器

**解决方案**：
1. 检查音频设备是否正确连接
2. 在Windows音频设置中确认设备为默认设备
3. 更新音频设备驱动程序
4. 尝试使用其他USB接口
5. 安装ASIO4ALL等专业音频驱动
